# aggelosmouzakitis-gr

Greek-market version of **aggelosmouzakitis.com** — positioned around *Σύμβουλος Ψυχικής Υγείας*
with a founder / tech-leadership background. It reuses the original site's design system
(colours, typography, components, sidebar layout) and adapts the copy for the Greek audience.

Separate from the `.com` site — no shared files.

## How it's built

Plain static HTML — no build step at serve time. The HTML is generated by a small Python
script so the shared navigation, footer and SEO `<head>` are defined once.

```bash
cd src
python3 build.py        # regenerates the HTML into ./site, copy over the repo root
```

Links are **relative and depth-aware**, so the site works both under a GitHub Pages
project sub-path (`/aggelosmouzakitis-gr/`) and later at a root custom domain.
Images reference the already-hosted assets on `aggelosmouzakitis.com`.

## Before the real launch — things to set

- **`BASE_URL`** in `src/build.py` — currently a placeholder (`https://aggelosmouzakitis.gr`).
  Set it to the final Greek domain; it drives `<link rel="canonical">`, Open Graph URLs and `sitemap.xml`.
- **Instagram URL** (`INSTAGRAM` in `src/build.py`) — placeholder; replace with the real handle.
- **Contact** — CTAs currently use a `mailto:` and the Burnout Diagnostic. Wire up a booking
  flow if wanted.
- **Analytics** — reuses the `.com` GA id; swap for a Greek property if you want separate tracking.

## Hosting (GitHub Pages)

Either:

1. **Actions (auto):** the workflow in `.github/workflows/pages.yml` deploys on every push to `main`
   and auto-enables Pages. In *Settings → Pages*, set **Source: GitHub Actions** if not already.
2. **Branch:** *Settings → Pages → Source: Deploy from a branch → `main` / `root`*.
   The `.nojekyll` file makes Pages serve the files as-is.
