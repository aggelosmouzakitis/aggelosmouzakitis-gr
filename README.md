# aggelosmouzakitis-gr

Greek-market version of **aggelosmouzakitis.com** — positioned around *Σύμβουλος Ψυχικής Υγείας*
with a founder / tech-leadership background. It reuses the original site's design system 1:1
(sidebar layout, colours, typography, components) and adapts the copy for the Greek audience.

Separate from the `.com` site. Photos, OG images and favicons are self-hosted (copied from the original).

Sidebar: **Αρχική · Μέθοδος · Υπηρεσίες** (dropdown → Executive Coaching / Burnout / Career Coaching).
**Σχετικά** and **Imposter Syndrome** live in the footer only. The **Burnout Diagnostic** is the
primary CTA (bottom of the sidebar, matching the original's CTA-box position).

## How it's built

Plain static HTML — no build step at serve time. The HTML is generated by a small Python
script so the shared sidebar, footer and SEO `<head>` are defined once.

```bash
cd src
python3 build.py        # regenerates ./site — copy its contents over the repo root
```

Links **and** assets are relative + depth-aware, so the site works both under a GitHub Pages
project sub-path (`/aggelosmouzakitis-gr/`) and later at a root custom domain, unchanged.

## Before the real launch — things to set

- **`BASE_URL`** in `src/build.py` — currently a placeholder (`https://aggelosmouzakitis.gr`).
  Drives `<link rel="canonical">`, Open Graph URLs and `sitemap.xml`.
- **Instagram URL** (`INSTAGRAM` in `src/build.py`) — placeholder; replace with the real handle.
- **Analytics** — reuses the `.com` GA id; swap for a Greek property if you want separate tracking.
- **Contact / booking** — CTAs use the Burnout Diagnostic + a `mailto:`. Wire up a booking flow if wanted.

## Hosting (GitHub Pages)

The workflow in `.github/workflows/pages.yml` deploys on every push to `main` and auto-enables Pages.
In *Settings → Pages*, Source should be **GitHub Actions**.
